# v0.13.0 进度记录 — 2026-04-30

## 今日完成项

### 1. P0 阻塞修复：`index` 外键错误（SQLite 787）
- **根因**：`save_modules()` 写入 `repo_modules_legacy`，该表保留 `REFERENCES repos(id)` FK 约束，但 `repos` 表在 v21 已废弃且为空
- **修复**：
  - `registry/knowledge.rs`：`save_modules()` 改为写入新 `repo_modules` 表（schema: `repo_id, module_name, module_type, module_path`）
  - `registry/migrate.rs`：v23 迁移 `DROP TABLE IF EXISTS repo_modules_legacy`
  - `init_db_at` 末尾增加无条件 `DROP TABLE IF EXISTS repos`，防止历史 `CREATE TABLE IF NOT EXISTS repos` 重建幽灵表
- **验证**：`devbase index .` 成功执行，无 FK 错误

### 2. Vault CLI 补全
- **新增子命令**：
  - `vault list [--tag <tag>]` — 列出所有 vault 笔记，支持按 tag 过滤
  - `vault read <path>` — 根据相对路径读取笔记完整内容（从文件系统读取）
  - `vault write <path> [--content <text>] [--title <title>]` — 写入或覆盖笔记；`--content -` 支持从 stdin 读取
- **实现文件**：`src/commands/simple.rs`、`src/main.rs`、`src/registry/vault.rs`（新增 `get_vault_note`）

### 3. OpLog MCP 工具暴露
- **新增工具**：`devkit_oplog_query`（Beta tier）
- **参数**：`limit`（1-100，默认 20）、`repo_id`（可选过滤）
- **返回**：事件数组，含 `id`, `event_type`, `repo_id`, `status`, `timestamp`, `duration_ms`, `details`
- **实现文件**：`src/mcp/tools/oplog.rs`、`src/mcp/mod.rs`、`src/mcp/tools/mod.rs`

### 4. `project_context` 代码结构增强
- **新增返回字段**：
  - `modules` — `repo_modules` 表中的 cargo target 列表（name, kind, path）
  - `symbols` — `code_symbols` 前 50 条（name, file, type, line, signature）
  - `calls` — `code_call_graph` 前 50 条（caller_file, caller, callee）
- **实现文件**：`src/mcp/tools/context.rs`
